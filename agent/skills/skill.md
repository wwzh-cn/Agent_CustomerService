# 技能清单 (Skills Directory)

## rag_summarize

**分类**: 检索类
**描述**: 从知识库检索相关文档并生成总结回答。适用于用户询问特定知识领域的问题，如扫地机器人推荐、故障排除等。
**参数**:   - query: 必填参数，用户查询文本，描述需要检索的知识主题
**示例**:   - "扫地机器人推荐"
  - "小户型适合哪些扫地机器人"
  - "rag_summarize(query='故障排除')"
**限制**:   - 依赖向量数据库中的知识库
  - 检索结果基于已有知识库，可能无法回答最新信息
  - 需要确保知识库已加载

## get_weather

**分类**: 查询类
**描述**: 查询中国城市实时天气信息，包括温度、湿度、风力、天气状况和数据发布时间。适用于用户询问天气情况时调用。
**参数**:   - city: 必填参数，中文城市名称，如'北京'、'上海市'
**示例**:   - "查询北京的天气"
  - "上海今天天气怎么样"
  - "get_weather(city='广州')"
**限制**:   - 仅支持中国城市
  - 需要网络连接
  - 数据来自高德天气API
  - 城市名称需准确，否则可能查询失败

## get_user_location

**分类**: 查询类
**描述**: 根据用户公网IP地址自动定位所在城市。适用于需要知道用户当前位置的场景。
**参数**: 无参数
**示例**:   - "我在哪个城市？"
  - "获取我的位置"
  - "定位用户所在城市"
**限制**:   - 依赖用户公网IP
  - 定位精度为城市级别
  - 需要网络连接
  - 如果IP定位失败返回'未知城市'

## get_user_id

**分类**: 数据类
**描述**: 获取用户的唯一标识符ID，用于用户识别和数据关联。
**参数**: 无参数
**示例**:   - "获取我的用户ID"
  - "用户标识是什么"
  - "get_user_id()"
**限制**:   - 返回随机选择的用户ID
  - ID来自预定义列表
  - 用于测试和演示目的

## get_current_month

**分类**: 数据类
**描述**: 获取当前月份信息，用于时间相关的数据查询和报告生成。
**参数**: 无参数
**示例**:   - "现在是几月份"
  - "获取当前月份"
  - "get_current_month()"
**限制**:   - 返回随机选择的月份
  - 月份来自预定义列表
  - 用于测试和演示目的

## fetch_external_data

**分类**: 数据类
**描述**: 从外部系统中获取指定用户在指定月份的使用记录数据。适用于生成用户使用报告的场景。
**参数**:   - user_id: 必填参数，用户ID，如'1001'、'1002'
  - month: 必填参数，月份，格式为'YYYY-MM'，如'2025-01'
**示例**:   - "获取用户1001在2025-01的使用记录"
  - "fetch_external_data(user_id='1002', month='2025-02')"
**限制**:   - 依赖外部数据文件
  - 如果用户或月份不存在返回空字符串
  - 数据文件路径在agent.yml中配置

## fill_context_for_report

**分类**: 控制类
**描述**: 触发中间件自动为报告生成的场景动态注入上下文信息，为后续提示词切换提供上下文。
**参数**: 无参数
**示例**:   - "准备生成报告"
  - "填充报告上下文"
  - "fill_context_for_report()"
**限制**:   - 无返回值，仅触发中间件操作
  - 用于报告生成前的上下文准备
  - 与报告提示词切换中间件配合使用
