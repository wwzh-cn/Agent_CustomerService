# 添加中期记忆（文件层持久化）执行计划

## 1. 背景与问题分析

### 1.1 短期记忆的局限性
已完成短期记忆（MemoryManager）实现，但面临以下限制：

1. **会话隔离**：MemoryManager 管理多会话短期记忆，但会话结束后记忆丢失
2. **缺乏持久化**：重启 Agent 后所有对话历史消失，无法积累知识
3. **知识无法复用**：有价值的对话洞察、学习成果无法跨会话共享
4. **上下文膨胀**：长期运行中，系统提示词缺乏动态更新机制

### 1.2 技术挑战
- **持久化策略**：如何设计文件结构平衡查询效率与维护成本
- **记忆整理**：如何从日常日志中自动提炼有价值信息
- **容量控制**：如何避免文件无限增长导致的系统提示词膨胀
- **集成兼容**：如何与现有 MemoryManager 无缝协作

## 2. 目标

### 2.1 核心目标
实现文件层持久化记忆系统，形成“会话层（短期）+ 文件层（中期）”双层记忆架构：

1. **持久化存储**：重要知识写入文件系统，跨会话持久保存
2. **智能整理**：自动从日常日志中提炼结构化知识
3. **容量可控**：严格限制记忆文件大小，避免系统提示词膨胀
4. **无缝集成**：与现有 MemoryManager 协同工作，不破坏已有功能

### 2.2 成功指标
- 重启 Agent 后能加载历史记忆，保持知识连续性
- MEMORY.md 文件大小稳定在 2000 字以内
- 记忆整理过程自动化，无需人工干预
- 系统启动时间增加 < 500ms
- 支持至少 100 天的日志积累

## 3. 技术方案

### 3.1 方案选择：文件层持久化架构

#### 3.1.1 核心组件
```
短期记忆层（MemoryManager）
    ↓（记录重要操作）
日常日志文件（按日期分割）
    ↓（定期整理）
长期记忆文件（MEMORY.md - 结构化知识）
    ↓（启动时加载）
Agent 系统提示词
```

#### 3.1.2 文件目录结构
```
memory/
├── IDENTITY.md      # Agent 身份定义（静态）
├── MEMORY.md        # 长期记忆库（动态更新，≤2000字）
├── TOOLS.md         # 工具配置和环境信息（静态）
└── logs/
    ├── 2026-03-31.md    # 每日工作日志（原始记录）
    ├── 2026-04-01.md    # 按日期组织，天然容量控制
    └── ...
```

#### 3.1.3 记忆生命周期
1. **实时记录**：重要操作、决策、学习结果追加到当日日志
2. **定期整理**：每隔 N 天（如 3 天）自动整理：
   - 读取近期日志
   - 提炼有价值信息（关键事实、决策、模式）
   - 更新 MEMORY.md 相应章节
   - 清理过时内容，保持 ≤2000 字
3. **启动加载**：每次启动加载 IDENTITY.md + MEMORY.md + TOOLS.md + 近期日志

### 3.2 推荐架构：FileMemory 管理器

#### 3.2.1 类设计
```python
class FileMemory:
    def __init__(self, base_dir: str = "./memory"):
        self.base_dir = Path(base_dir)
        self.base_dir.mkdir(exist_ok=True)
    
    def load_identity(self) -> str:
        """启动时加载身份文件（IDENTITY.md + MEMORY.md + TOOLS.md）"""
    
    def load_recent_logs(self, days: int = 2) -> str:
        """加载最近 N 天的日志内容"""
    
    def append_log(self, content: str):
        """追加内容到今日日志文件"""
    
    def update_memory(self, key: str, value: str):
        """更新 MEMORY.md 中的指定条目"""
    
    def consolidate_memory(self, days_to_review: int = 7):
        """记忆整理：从近期日志提炼知识到 MEMORY.md"""
```

#### 3.2.2 记忆整理策略
```python
class MemoryConsolidator:
    STRATEGIES = {
        "extract_facts": "提取关键事实和数值",
        "summarize_decisions": "总结决策过程和结果",
        "identify_patterns": "识别重复出现的模式和规律",
        "flag_unsolved": "标记未解决的问题和待办事项"
    }
    
    def consolidate(self, log_contents: List[str]) -> Dict[str, str]:
        """从日志内容提炼结构化知识"""
```

### 3.3 与现有系统集成设计

#### 3.3.1 集成点
1. **Agent 初始化时**：加载文件记忆到系统提示词
2. **重要操作后**：调用 `append_log()` 记录到日志
3. **定期任务**：定时触发 `consolidate_memory()` 整理记忆
4. **手动更新**：通过 `update_memory()` 直接添加重要知识

#### 3.3.2 配置设计

## 5. 风险与应对

### 5.1 技术风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 文件操作导致性能下降 | 中 | 中 | 异步日志写入；批量操作；文件缓存 |
| 记忆整理消耗过多资源 | 低 | 中 | 限制整理频率；可选关闭 LLM 辅助整理 |
| 文件损坏或格式错误 | 低 | 高 | 实现文件校验和恢复机制；定期备份 |
| 系统提示词超出限制 | 中 | 高 | 严格限制 MEMORY.md 大小；动态截断策略 |
| 并发访问冲突 | 中 | 中 | 文件锁机制；原子写入操作 |

### 5.2 实现风险
- **与现有 MemoryManager 冲突**：功能重叠或接口不一致
- **应对**：明确职责划分，MemoryManager 管会话流，FileMemory 管持久化

### 5.3 维护风险
- **记忆文件膨胀**：长期运行后文件过大
- **应对**：严格的容量控制；定期归档旧日志；提供清理工具
