# 高德MCP集成需求分析

**文档版本**: 1.0
**创建日期**: 2026-03-29
**项目状态**: 技术验证通过，准备实施

## 1. 项目概述

### 1.1 背景
智扫通机器人智能客服系统当前使用高德地图API实现天气查询(`get_weather`)和用户定位(`get_user_location`)功能。这些功能直接嵌入在`agent/tools/agent_tools.py`中，通过HTTP调用高德API。

### 1.2 目标
将现有高德API调用迁移至高德官方MCP服务器，实现：
- **工具解耦**：移除直接HTTP调用逻辑
- **统一协议**：使用标准化MCP协议
- **维护简化**：依赖高德官方维护的MCP服务

### 1.3 约束条件
- 个人项目，流程尽量简单
- 保持现有接口完全兼容
- 最小化代码改动

### 2.2 现有工具接口

#### `get_weather(city: str) -> str`
- **位置**: `agent/tools/agent_tools.py:88-118`
- **功能**: 查询指定城市天气
- **返回值示例**: "城市北京天气为晴，气温25摄氏度，空气湿度60%，东南风3级，数据发布时间2025-01-01 12:00:00。"
- **错误处理**: "未提供城市名称，无法查询天气" / "城市{city}天气查询失败，请稍后重试"

#### `get_user_location() -> str`
- **位置**: `agent/tools/agent_tools.py:122-159`
- **功能**: 获取用户所在城市
- **返回值示例**: "北京市" / "上海市"
- **错误处理**: "未知城市"

## 3. 需求目标

### 3.1 功能需求
1. **完全替换**: 用高德MCP工具替代现有`get_weather`和`get_user_location`
2. **接口兼容**: 保持相同函数签名和返回值格式
3. **零感知切换**: 用户无需感知底层实现变更

### 3.2 非功能需求
1. **简单性**: 个人项目，避免复杂架构
2. **可维护性**: 代码清晰，易于理解
3. **可测试性**: 便于验证功能正确性
4. **稳定性**: 错误处理健壮，适当降级

## 4. 技术方案

### 4.1 方案选择：直接MCP集成
**不**自研MCP服务器，**直接使用**高德官方MCP服务。

### 4.2 架构设计
```

├── rag_summarize
├── mcp_weather          ← 高德MCP天气工具（适配层）

├── fetch_external_data
└── fill_context_for_report
```

### 4.3 技术栈
- **MCP客户端**: `langchain_mcp_adapters`
- **传输协议**: HTTP/SSE
- **配置管理**: 复用现有`config/agent.yml`
- **错误处理**: 保持现有错误消息格式

## 5. 接口兼容性分析

### 5.1 潜在挑战
| 方面 | 现有实现 | 高德MCP可能方案 |
|------|----------|----------------|
| **工具名称** | `get_weather` | `weather_query`, `amap_weather`, `weather_info` |
| **工具名称** | `get_user_location` | `ip_location`, `geolocation`, `location_by_ip` |
| **天气参数** | `city: str` | 可能需要`city`、`adcode`或经纬度 |
| **定位参数** | 无参数 | 可能需要`ip`参数 |
| **返回格式** | 纯字符串 | 可能返回结构化JSON |

### 5.2 兼容性保障策略
1. **适配层设计**: 必要时添加简单适配函数
2. **工具发现机制**: 运行时检查可用工具并选择匹配项
3. **降级处理**: MCP不可用时保留回退能力

## 8. 风险评估与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| **高德MCP工具不匹配** | 中 | 高 | 添加适配层，保持接口兼容 |
| **网络延迟增加** | 低 | 中 | SSE连接复用，超时控制 |
| **API Key权限不足** | 低 | 高 | 验证现有Key的MCP权限 |
| **工具接口变更** | 低 | 中 | 封装适配函数，隔离变更 |
| **MCP服务不可用** | 低 | 高 | 保留原有实现作为备选 |
